1.- Configurar el entorno
Software y frameworks a utilizar

Para el desarrollo del prototipo de la Plataforma de Gestión de Excedentes Alimentarios, se utilizará un entorno basado en Docker + n8n, ya que permite construir flujos funcionales de manera visual y automatizada, sin requerir una implementación completa desde cero.

Software / herramienta	Versión utilizada	Uso en el proyecto
Docker Desktop	4.x o superior	Ejecutar n8n en un contenedor.
Docker Engine	27.x o superior	Motor de ejecución de contenedores.
Docker Compose	Incluido en Docker Desktop	Levantar el entorno mediante archivo docker-compose.yml.
n8n	1.x	Crear flujos de automatización para publicación, alerta y reserva de donaciones.
Visual Studio Code	1.80 o superior	Editar archivos del proyecto y documentación.
Git Bash	2.x o superior	Ejecutar comandos Git.
GitHub	Plataforma web	Control de versiones y trabajo colaborativo.
Navegador web	Chrome / Edge	Acceso a n8n desde localhost:5678.
Instalación de dependencias

En este proyecto no se requiere instalar dependencias manualmente con npm, pip u otro gestor, ya que n8n se ejecuta directamente mediante Docker.

La única dependencia principal es tener instalado y funcionando:

docker --version

También se verifica Docker Compose con:

docker compose version
Manual de instalación
Instalar Docker Desktop en el computador.
Instalar Git Bash.
Instalar Visual Studio Code.
Crear la carpeta del proyecto:
mkdir Proyecto-Desperdicio-de-Alimentos
cd Proyecto-Desperdicio-de-Alimentos
Crear el archivo docker-compose.yml con la siguiente configuración:
services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n-desperdicio-alimentos
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - TZ=America/Santiago
      - GENERIC_TIMEZONE=America/Santiago
      - N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true
    volumes:
      - n8n_data:/home/node/.n8n
      - ./n8n/workflows:/workflows

volumes:
  n8n_data:
Levantar el entorno con Docker:
docker compose up -d
Verificar que el contenedor esté activo:
docker ps
Ingresar a n8n desde el navegador:
http://localhost:5678
Verificación del entorno

Para comprobar que el entorno funciona correctamente, se realizaron las siguientes pruebas:

Prueba 1: Verificación de Docker

Comando ejecutado:

docker ps

Resultado esperado:

El contenedor n8n-desperdicio-alimentos aparece en estado Up.
Prueba 2: Acceso a n8n

Se ingresó desde el navegador a:

http://localhost:5678

Resultado esperado:

La interfaz de n8n carga correctamente.
Prueba 3: Workflow de prueba

Se creó un workflow básico en n8n con los siguientes nodos:

Manual Trigger → Set

En el nodo Set se agregaron datos de prueba:

{
  "producto": "Pan",
  "cantidad": "10 kg",
  "categoria": "Panadería",
  "estado": "Disponible"
}

Resultado esperado:

El workflow se ejecuta correctamente y muestra los datos configurados.
Conclusión de la configuración

El entorno fue configurado correctamente utilizando Docker, n8n, Visual Studio Code, Git Bash y GitHub. La ejecución del contenedor fue verificada mediante comandos de Docker, y el acceso a n8n fue validado desde el navegador. Además, se creó un workflow de prueba para confirmar que la plataforma permite construir flujos funcionales, dejando el entorno listo para desarrollar el prototipo del sistema de gestión de excedentes alimentarios.
